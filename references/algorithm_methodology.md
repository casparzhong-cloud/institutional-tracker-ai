## 1. 资金流市值归一化 (Kang 2025)

### 问题

原始算法使用绝对净流入金额评分。这导致大市值股票（如宁德时代，市值过万亿）天然获得更高分，
小市值成长股（如源杰科技，市值百亿级）即使有相同比例的资金流入，分数也远低于大盘股。

### 解决方案

```python
norm_ratio = net_main_yi / mv_yi * 100  # 净流入占市值百分比
if norm_ratio > 0:
    base = min(90, 20 + 35 * math.log10(max(0.001, norm_ratio) * 100))
```

用对数尺度将净流入/市值比例映射到评分：
- 小市值股+1亿净流入 → 高分
- 大市值股+1亿净流入 → 中等分

### 论文依据

Kang et al. (2025) "Institutional Trading and Stock Market Microstructure":
机构持仓变动需按标的自由流通市值归一化，否则统计检验会被市值效应污染。

## 2. 量价逻辑反转

### 问题

传统技术分析认为"放量上涨=看涨"。但A股实证数据（量价趋势2023、国金2022）显示：
- A股散户主导市场中，放量上涨往往是**散户追高的末端**，预示反转
- 缩量横盘反而是**机构安静吸筹的典型模式**

### 改进

| 模式 | v7分数 | v8分数 | 原因 |
|------|--------|--------|------|
| 放量温和上涨 | +65 | +30 | 降权：可能是散户追高 |
| 缩量横盘 | +35 | +55 | 加权：安静吸筹 |
| 缩量小跌 | +20 | +45 | 加权：洗盘吸筹 |
| 放量大涨后缩量 | 无 | +50 | 新增：主力控盘不出货 |

## 3. 散户反向过滤

### 论文依据

- 上交大(2019): A股散户交易行为的信息含量为负（散户净买入后1-20日显著跑输）
- Ravina (2023): 散户订单流与未来收益呈显著负相关

### 实现

```python
if net_sm_yi > 0 and net_main_yi > 0:
    # 散户和主力同向买入 → FOMO末端，降权
    if net_sm_yi > abs(net_main_yi) * 0.5:
        score -= 15
elif net_sm_yi < 0 and net_main_yi > 0:
    # 主力买入+散户卖出 = 经典吸筹模式，加分
    score += 10
```

## 4. 卖出信号的砍除 (v11)

### 数据支撑

16503条全周期信号回测：
- 买入信号胜率: 55.3% → P=0.007 → 统计显著
- 卖出信号正确率: 41.9% → 比抛硬币还差 → 完全无效

### 理论支撑

Campbell et al. (2009): "预测市场顶部比预测趋势难一个数量级"

### 解决方案

- 不再输出 `DISTRIBUTION` 状态
- 改为 `CAUTION`（风险警示），不构成交易信号
- 依赖机械止损: 持有10日 + 回撤-15%止损

## 5. BEAR市禁入 (v10)

### 逻辑

回测发现：在 BEAR 环境下的买入信号胜率接近随机甚至低于50%。
市场整体下行时，个股的"吸筹信号"很可能是"下跌中继"的假信号。

### 实现

```python
if regime == "BEAR":
    accumulation_th = 999  # 任何分数都不可能>=999，等于禁止买入
    markup_th = 999
```

## 6. 日内分钟线分析 (v11新增)

### 四个维度

1. **尾盘放量**：占比>35% → +20分（Campbell 2009: 机构集中尾盘操作）
2. **开盘抢筹/出货**：开盘30分钟放量+全天方向判断
3. **成交均匀度**：CV(变异系数)<0.4 → +10分（VWAP算法交易特征）
4. **VWAP偏离**：收盘>VWAP → 买方主导

### 权重

日内分钟线加成占总分的15%（权重较低，因为数据质量不如日线稳定）。

## 7. 外部情绪聚合

### 5个数据源

1. YouTube 财经频道（RSS免费获取）
2. 东方财富股吧情绪
3. 东方财富/新浪财经资讯关键词
4. 东方财富热股趋势
5. 东方财富人气指数（散户过热代理）

### 集成方式

- 聚合后的情绪分数(-100~+100)作为市场环境判断的第10个指标
- 权重8%（弱信号，r=0.069 but p=0.028）
- 个股过热信号 `detect_stock_overheat()` 可作为信号惩罚因子
